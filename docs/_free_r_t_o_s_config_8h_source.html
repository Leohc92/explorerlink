113 /* Info at: http://www.freertos.org/a00110.html#kernel_priority */
114 /* The priority for the FreeRTOS kernel interrupt. This must be the lowest
115  * priority so that ISRs can preempt the kernel. */
116 #define configKERNEL_INTERRUPT_PRIORITY ( 7 << 5 )
117 /* The highest priority allowable for interrupts whose ISRs contain FreeRTOS
118  * API calls.
119  *
120  * When the FreeRTOS kernel enters a critical section, it needs to ensure that
121  * no API calls can be made that could corrupt kernel data. ISRs that contain
122  * API calls could cause this to occur, so the kernel needs a way to ensure
123  * that these interrupts are masked. configMAX_SYSCALL_INTERRUPT_PRIORITY
124  * introduces a threshold value for interrupt priorities, at or below which
125  * the kernel CAN mask interrupts during a critical section. Thus, by trading
126  * away a bit of response time, ISRs are able to make use of API functions.
127  * Interrupts whose ISRs do not need API functions and can't afford the latency
128  * penalty should be given priorities above this value so that they will
129  * operate with no RTOS interference.
130  *
131  * FreeRTOS maintains a separate interrupt-safe API to ensure interrupt entry
132  * is as fast and as simple as possible. ISRs must only utilize these API
133  * calls, which end with "FromISR". */
134 #define configMAX_SYSCALL_INTERRUPT_PRIORITY ( 5 << 5 )
